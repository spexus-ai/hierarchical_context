# Daily Run Example

## Вход
- новые PR
- обновления метрик
- один инцидент
- один market intelligence signal

## Ожидаемый проход
1. L4 собирает evidence из всех доступных источников
2. L4 публикует evidence в knowledge git repository по проектным папкам
3. L3 создаёт/обновляет change streams с привязкой к owning L2
4. L2 обновляет affected ownership zones
5. L1 пересобирает affected domains и cross-project flows
6. L0 обновляет global picture портфеля и стратегических целей
7. OpenClaw получает задачу из L3 и публикует execution evidence обратно в L4
