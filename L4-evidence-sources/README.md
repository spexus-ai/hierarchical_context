# L4 — Evidence / Sources

## Что это

Нижний уровень контекста — первоисточники.

Это единственный слой, который должен содержать фактические артефакты:
- PR
- ADR
- коммиты
- дашборды
- тикеты
- логи инцидентов
- отчёты тестов
- обсуждения
- trace ИИ-агентов
- другие подтверждающие материалы

L4 может вестись в отдельном knowledge git repository, куда пишут несколько команд и агентов.

## Задачи уровня

- быть источником истины по фактам;
- позволять максимальное погружение;
- поддерживать трассируемость решений;
- давать материал для summary верхних уровней.
- обеспечивать прозрачность по всем проектам портфеля;
- накапливать знания так, чтобы они были доступны для кросс-переноса между проектами.

## Какие решения принимаются

На этом уровне обычно не принимаются управленческие решения сами по себе.
Здесь принимаются решения о корректности фактов:
- подтверждён ли эффект;
- действительно ли есть регрессия;
- есть ли доказательство риска;
- можно ли доверять summary верхнего уровня.

## Правила

- любой summary выше должен ссылаться на evidence здесь;
- без ссылки на evidence изменение считается недостаточно обоснованным;
- артефакты должны быть датированы и иметь owner / source.
- один проект = одна папка в knowledge repo;
- одна запись evidence = один файл;
- `journal.md` проекта допустим как индекс, но не как primary storage;
- если evidence не удаётся сразу отнести к проекту, оно публикуется в `triage/inbox`;
- общие потоки знаний можно вести в `shared-intelligence`, например:
  - `market-intelligence`
  - `customer-intelligence`
  - `partner-vendor-intelligence`
  - `regulatory-intelligence`
  - `security-intelligence`
  - `operations-intelligence`
